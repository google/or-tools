26// Solve the input MIP model with the SAT solver.
27//
28// If possible, std::move the request into this function call to avoid a copy.
29//
30// If you need to change the solver parameters, please use the
31// EncodeSatParametersAsString() function below to set the request's
32// solver_specific_parameters field.
33//
34// The optional interrupt_solve can be used to interrupt the solve early. It
35// must only be set to true, never reset to false. It is also used internally by
36// the solver that will set it to true for its own internal logic. As a
37// consequence the caller should ignore the stored value and should not use the
38// same atomic for different concurrent calls.
39//
40// The optional logging_callback will be called when the SAT parameter
41// log_search_progress is set to true. Passing a callback will disable the
42// default logging to INFO. Note though that by default the SAT parameter
43// log_to_stdout is true so even with a callback, the logs will appear on stdout
44// too unless log_to_stdout is set to false. The enable_internal_solver_output
45// in the request will act as the SAT parameter log_search_progress.
46//
47// The optional solution_callback will be called on each intermediate solution
48// found by the solver. The solver may call solution_callback from multiple
49// threads, but it will ensure that at most one thread executes
50// solution_callback at a time.
51absl::StatusOr<MPSolutionResponse> SatSolveProto(
52 MPModelRequest request, std::atomic<bool>* interrupt_solve = nullptr,
53 std::function<void(const std::string&)> logging_callback = nullptr,
54 std::function<void(const MPSolution&)> solution_callback = nullptr);
55
56// Returns a string that should be used in MPModelRequest's
57// solver_specific_parameters field to encode the SAT parameters.
58//
59// The returned string's content depends on the version of the proto library
60// that is linked in the binary.
61//
62// By default it will contain the textual representation of the input proto.
63// But when the proto-lite is used, it will contain the binary stream of the
64// proto instead since it is not possible to build the textual representation in
65// that case.
66//
67// The SatSolveProto() function will test if the proto-lite is used and expect a
68// binary stream when it is the case. So in order for your code to be portable,
69// you should always use this function to set the specific parameters.
70std::string EncodeSatParametersAsString(const sat::SatParameters& parameters);

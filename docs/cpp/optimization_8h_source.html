49// All the Solve*() functions below reuse the SatSolver::Status with a slightly
50// different meaning:
51// - FEASIBLE: The problem has been solved to optimality.
52// - INFEASIBLE: Same meaning, the decision version is already unsat.
53// - LIMIT_REACHED: we may have some feasible solution (if solution is
54// non-empty), but the optimality is not proven.

56// Implements the "Fu & Malik" algorithm described in:
57// Zhaohui Fu, Sharad Malik, "On solving the Partial MAX-SAT problem", 2006,
58// International Conference on Theory and Applications of Satisfiability
59// Testing. (SAT’06), LNCS 4121.
60//
61// This algorithm requires all the objective weights to be the same (CHECKed)
62// and currently only works on minimization problems. The problem is assumed to
63// be already loaded into the given solver.
64//
65// TODO(user): double-check the correctness if the objective coefficients are
66// negative.
67SatSolver::Status SolveWithFuMalik(LogBehavior log,
68 const LinearBooleanProblem& problem,
69 SatSolver* solver,
70 std::vector<bool>* solution);
71
72// The WPM1 algorithm is a generalization of the Fu & Malik algorithm to
73// weighted problems. Note that if all objective weights are the same, this is
74// almost the same as SolveWithFuMalik() but the encoding of the constraints is
75// slightly different.
76//
77// Ansotegui, C., Bonet, M.L., Levy, J.: Solving (weighted) partial MaxSAT
78// through satisﬁability testing. In: Proc. of the 12th Int. Conf. on Theory and
79// Applications of Satisﬁability Testing (SAT’09). pp. 427-440 (2009)
80SatSolver::Status SolveWithWPM1(LogBehavior log,
81 const LinearBooleanProblem& problem,
82 SatSolver* solver, std::vector<bool>* solution);

84// Solves num_times the decision version of the given problem with different
85// random parameters. Keep the best solution (regarding the objective) and

93// Starts by solving the decision version of the given LinearBooleanProblem and
94// then simply add a constraint to find a lower objective that the current best
95// solution and repeat until the problem becomes unsat.
96//
97// The problem is assumed to be already loaded into the given solver. If
98// solution is initially a feasible solution, the search will starts from there.
99// solution will be updated with the best solution found so far.
100SatSolver::Status SolveWithLinearScan(LogBehavior log,
101 const LinearBooleanProblem& problem,
102 SatSolver* solver,
103 std::vector<bool>* solution);

118// Model-based API, for now we just provide a basic algorithm that minimizes a
119// given IntegerVariable by solving a sequence of decision problem by using

138// Same as MinimizeIntegerVariableWithLinearScanAndLazyEncoding() but use

215// Generalization of the max-HS algorithm (HS stands for Hitting Set). This is

219// See, Jessica Davies and Fahiem Bacchus, "Solving MAXSAT by Solving a

221// http://www.cs.toronto.edu/~jdavies/daviesCP11.pdf"
222//
223// Note that an implementation of this approach won the 2016 max-SAT competition
224// on the industrial category, see
225// http://maxsat.ia.udl.cat/results/#wpms-industrial
226//
